# 埋点分析系统架构

本文档描述 GateFlow Tracker 数据采集系统的整体架构设计。

## 设计目标

- **独立部署**: 与 AB 实验服务解耦，可独立运维
- **全面采集**: 覆盖曝光、点击、停留时长、路径等维度
- **高性能**: 支持高并发写入，批量上报
- **可扩展**: 数据模型支持灵活扩展，服务端无状态水平扩展

## 数据采集维度

### 页面级分析

| 指标 | 采集方式 |
|------|---------|
| PV/UV | SPA 路由切换监听 |
| 跳出率 | 单页会话标识 |
| 停留时长 | visibilitychange API |
| 来源 | document.referrer |

### 元素级分析

| 指标 | 采集方式 |
|------|---------|
| 曝光率 | Intersection Observer |
| 点击率 | 事件代理 + data-track 属性 |
| 转化漏斗 | 会话级序列聚合 |

### 归因分析

| 维度 | 来源 |
|------|------|
| 来源渠道 | utm_source |
| 营销媒介 | utm_medium |
| 营销活动 | utm_campaign |
| 首次/末次触点 | 时间窗口归因 |

## 事件类型

| 类型 | 说明 | 上报策略 |
|------|------|---------|
| `page_view` | 页面浏览 | 批量上报 |
| `click` | 点击事件 | 立即上报 |
| `exposure` | 元素曝光 | 立即上报 |
| `scroll` | 滚动事件 | 批量上报 |
| `stay` | 停留事件 | 批量上报 |
| `custom` | 自定义事件 | 按配置 |

## 可靠性设计

| 机制 | 实现 | 说明 |
|------|------|------|
| **熔断保护** | Resilience4j CircuitBreaker | ClickHouse 故障时自动熔断，事件写入 DLQ |
| **死信队列** | Redis DLQ (7天 TTL) | 失败事件持久化，定时重放 |
| **去重** | Redis SET NX (5分钟窗口) | 基于 eventId 去重，Redis 故障时降级 |
| **客户端离线队列** | IndexedDB | 网络断开时暂存，恢复后批量重发 |
