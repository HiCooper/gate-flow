# 埋点系统产品指南

本文档面向产品经理和运营人员，介绍如何使用 Tracker 埋点系统进行用户行为分析。

## Tracker 是什么

Tracker 是 GateFlow 平台的数据采集组件，帮助你了解：

- 用户在做什么（点击、浏览、停留）
- 用户从哪个渠道来
- 用户的行为路径是什么
- 实验组用户表现如何

## 核心指标

### 页面级

| 指标 | 说明 | 业务价值 |
|------|------|---------|
| PV | 页面被访问次数 | 衡量内容受欢迎程度 |
| UV | 独立用户数 | 衡量真实用户覆盖 |
| 跳出率 | 单页访问就离开的比例 | 评估落地页质量 |
| 平均停留时长 | 用户在页面的平均时间 | 评估内容吸引力 |

### 元素级

| 指标 | 说明 | 业务价值 |
|------|------|---------|
| 曝光量 | 元素被看到的次数 | 评估展示位置效果 |
| 点击量 | 元素被点击的次数 | 评估元素吸引力 |
| CTR | 点击量 / 曝光量 | 评估元素效果 |
| 转化率 | 完成目标行为的用户比例 | 评估转化效率 |

### 路径分析

| 指标 | 说明 |
|------|------|
| 路径序列 | 用户访问页面的顺序 |
| 关键节点到达率 | 到达关键页面的比例 |
| 转化漏斗 | 每一步的流失情况 |

## 事件类型

| 类型 | 触发方式 | 说明 |
|------|---------|------|
| `page_view` | 自动采集 | 进入页面时 |
| `scroll` | 自动采集 | 滚动超过阈值时 |
| `click` | 需标记 | 按钮、链接点击 |
| `exposure` | 需标记 | 重要元素曝光 |

## 埋点路径

格式：`a_{应用}.b_{页面}.c_{区块}.d_{点位}`

示例：
- `a_home.b_home.c_banner.d_slide_1` → 首页横幅第1张轮播
- `a_app.b_product.c_action.d_buy_btn` → 商品页购买按钮

## 创建实验指标

在 A/B 实验中基于埋点数据创建自定义指标：

```
指标名称：购买按钮点击率
计算公式：点击量(spmd="d_buy_btn") / 曝光量(spmd="d_buy_btn")
事件类型：click / exposure
筛选条件：spmb = "b_product"
```

## 数据时效

| 类型 | 延迟 |
|------|------|
| 实时数据 | ~1 分钟 |
| 聚合数据 | ~5 分钟 |

## 常见问题

**数据丢失怎么办？** 系统内置离线缓存（IndexedDB）、DLQ 死信队列、熔断保护三重保障。

**数据保留多久？** 事件明细和会话数据保留 90 天。

**如何关联实验？** 通过 user_id 自动关联，实验分组信息内嵌在事件的 experimentTags 中。
