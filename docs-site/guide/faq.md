# 常见问题

本文档汇总使用 GateFlow 时的常见问题。

## 实验相关

### Q: 实验需要运行多久？

A: 取决于流量和预期效应大小，通常建议：
- 至少运行 1-2 个完整业务周期
- 样本量达到统计计算的要求
- 避免在周末/节假日期间开始

### Q: 如何判断实验结果可靠？

A: 检查以下条件：
1. 统计显著性 (p < 0.05)
2. 样本量是否充足
3. 护栏指标是否正常
4. 无 SRM 问题

### Q: 可以中途修改实验配置吗？

A: 不建议。实验期间的任何修改都可能导致：
- 数据污染
- 结果不可信
- 分析困难

如必须修改，建议结束当前实验，重新开启新实验。

### Q: 用户中途切换变体会怎样？

A: GateFlow 使用确定性分桶，同一用户始终分到同一变体。如果需要重置分配，可使用白名单功能。

## 技术相关

### Q: SDK 如何获取实验配置？

A: SDK 启动时自动拉取配置，并定时轮询更新：
```
GET /api/v1/config/fetch
```

### Q: 分桶结果在各端一致吗？

A: 是的。各端 SDK 使用相同的 MurmurHash3 算法，确保分桶结果一致。

### Q: 网络异常时 SDK 如何处理？

A: SDK 优先使用本地缓存的配置，保证服务可用性。

### Q: 如何处理用户隐私？

A: 请确保：
- 遵守相关隐私法规
- 用户 ID 不包含敏感信息
- 事件数据脱敏处理
