# GateFlow Tracker 产品使用指南

> 面向产品经理、运营人员的埋点方案说明

---

## 1. 什么是 Tracker？

Tracker（行为采集系统）是 GateFlow 平台的数据采集组件，用于收集用户在产品中的行为数据，帮助你了解：
- 用户在做什么？（点击、浏览、停留）
- 用户是如何到达的？（从哪个渠道来）
- 用户的行为路径是什么？（访问顺序、转化漏斗）
- 实验组用户表现如何？（与 A/B 实验关联）

## 2. 核心指标说明

### 2.1 页面级指标

| 指标 | 说明 | 业务价值 |
|------|------|----------|
| **PV（页面浏览量）** | 页面被访问的次数 | 衡量内容受欢迎程度 |
| **UV（独立访客数）** | 独立的用户数量 | 衡量真实用户覆盖 |
| **跳出率** | 只访问一个页面就离开的比例 | 评估落地页质量 |
| **平均停留时长** | 用户在页面的平均停留时间 | 评估内容吸引力 |
| **页面退出率** | 从该页面离开的比例 | 识别流失点 |

### 2.2 元素级指标

| 指标 | 说明 | 业务价值 |
|------|------|----------|
| **曝光量** | 元素被用户看见的次数 | 评估展示位置效果 |
| **曝光率** | 曝光量 / 页面访问量 | 评估曝光充分性 |
| **点击量** | 元素被点击的次数 | 评估元素吸引力 |
| **点击率（CTR）** | 点击量 / 曝光量 | 评估元素效果 |
| **转化率** | 完成目标行为的用户比例 | 评估整体转化效率 |

### 2.3 路径分析指标

| 指标 | 说明 | 业务价值 |
|------|------|----------|
| **路径序列** | 用户访问页面的顺序 | 了解真实用户旅程 |
| **关键节点到达率** | 用户到达关键页面的比例 | 评估路径设计 |
| **转化漏斗** | 每一步的流失情况 | 识别优化机会 |
| **归因路径** | 用户转化前经过的触点 | 评估渠道贡献 |

## 3. 如何使用埋点数据？

### 3.1 创建实验指标

在 A/B 实验中，你可以基于埋点数据创建自定义指标：

```
实验目标：提升「立即购买」按钮的点击率

定义方式：
- 指标名称：购买按钮点击率
- 计算公式：点击量（spmd = "d_buy_btn"） / 曝光量（spmd = "d_buy_btn"）
- 事件类型：click / exposure
- 筛选条件：spmb = "b_product"（商品详情页）
```

### 3.2 分析用户行为

场景示例：
- 首页 Banner 的曝光和点击情况
- 商品卡片区域的用户停留时长分布
- 用户从进入商品页到下单的完整路径

### 3.3 渠道效果评估

通过 UTM 参数追踪不同渠道的用户质量：

| 渠道 | UV | 转化率 | 平均停留时长 |
|------|-----|--------|-------------|
| 搜索引擎 | 5,000 | 3.2% | 45s |
| 社交媒体 | 3,200 | 2.1% | 38s |
| 直接访问 | 1,800 | 4.5% | 62s |

## 4. 埋点事件类型

### 4.1 自动采集事件

系统会自动采集以下事件，无需额外配置：

| 事件类型 | 触发时机 | 包含数据 |
|----------|----------|----------|
| `page_view` | 进入页面时 | 页面URL、标题、来源、停留时长 |
| `scroll` | 滚动超过阈值时 | 滚动深度（25%/50%/75%/100%） |

### 4.2 需要标记的事件

需要在代码中标记的采集点，通过埋点路径标识：

| 事件类型 | 使用场景 | 埋点路径示例 |
|----------|----------|--------------|
| `click` | 按钮、链接点击 | `a_app.b_product.c_action.d_buy_btn` |
| `exposure` | 重要元素曝光 | `a_home.b_home.c_banner.d_slide_1` |
| `custom` | 自定义业务事件 | `a_app.b_checkout.c_payment.d_wechat_pay` |

**埋点路径结构**：`a_{应用}.b_{页面}.c_{区块}.d_{点位}`

存储字段：`spma`(应用) / `spmb`(页面) / `spmc`(区块) / `spmd`(点位)

| 层级 | 前缀 | 字段 | 说明 | 示例 |
|------|------|------|------|------|
| 应用 | `a_` | spma | 业务应用 | `a_app` 主站、`a_ds` 数据平台 |
| 页面 | `b_` | spmb | 页面标识 | `b_home` 首页、`b_product` 商品详情 |
| 区块 | `c_` | spmc | 功能区块 | `c_banner` 横幅、`c_action` 操作区 |
| 点位 | `d_` | spmd | 具体元素 | `d_buy_btn` 购买按钮、`d_slide_1` 第1张图 |

示例路径解读：

- `a_home.b_home.c_banner.d_slide_1` → 应用首页 > 页面首页 > 横幅 > 第1张轮播
- `a_app.b_product.c_action.d_add_cart` → 主应用 > 商品详情 > 操作区 > 加购按钮

## 6. 数据查看方式

### 6.1 DS 平台（数据探索）

在 DS 平台中，你可以：
- 查看实时指标看板
- 分析用户行为路径
- 对比不同实验组的数据

### 6.2 自定义分析

支持 SQL 查询，方便进行深度分析：

```sql
-- 查看某页面不同实验组的用户行为
SELECT
    variant,
    COUNT(DISTINCT user_id) as users,
    AVG(stay_duration) as avg_stay,
    COUNT(CASE WHEN event_type = 'click' THEN 1 END) as clicks
FROM events
WHERE page_url = '/product/detail'
  AND timestamp > '2026-05-01'
GROUP BY variant
```

## 7. 常见问题

### Q1: 如何知道用户是否看到了某个 Banner？
使用曝光事件：在 Banner 元素上添加 `data-exposure="banner_001"`，系统会自动采集该 Banner 的曝光情况。

### Q2: 如何追踪用户从哪个渠道来？
通过 UTM 参数自动采集：系统在用户访问时会自动解析 URL 中的 utm_source、utm_medium、utm_campaign 等参数。

### Q3: 实验组用户的数据如何与分析系统关联？
通过 User ID 关联：用户进入实验后，其实验分组信息会与行为数据自动关联，方便后续分析不同实验组用户的行为差异。

### Q4: 埋点数据多久可以看到？

- 实时数据：延迟约 1 分钟
- 聚合数据：延迟约 5 分钟

### Q5: 如果数据采集出现问题怎么办？

系统内置多重可靠性保障：

1. **离线缓存**：用户设备离线时，事件会暂存在本地（IndexedDB），网络恢复后自动补发
2. **服务端重试**：写入 ClickHouse 失败时，事件会进入 Dead Letter Queue（DLQ），后续自动重放
3. **熔断保护**：ClickHouse 故障时不会导致事件丢失，数据会在 DLQ 中等待恢复后处理

### Q6: 数据会保留多久？

- 事件明细数据：90 天（自动清理）
- 会话汇总数据：90 天
- 物化视图聚合：实时更新，无独立保留期

如需更长的保留期，请联系技术团队配置。

### Q7: 如何识别数据质量问题？

在 DS 平台的数据探索页面，可以关注以下指标：

| 异常现象 | 可能原因 | 排查方式 |
|---------|---------|---------|
| 某天数据量突然下降 | SDK 上线问题 / 服务故障 | 检查 DLQ 积压量 |
| 事件延迟到达 | 网络问题 / 服务过载 | 检查 P99 延迟 |
| 用户行为突然中断 | 会话超时 / 客户端崩溃 | 检查会话完成率 |
