设计说明
1. 客户端层：业务 App 使用 GateFlow SDK 的入口，通过 React Hook 与 SDK 交互
2. SDK 层：提供跨平台抽象，内部调用 Native Module 完成分桶和本地缓存
3. 网关层：统一的请求入口，负责鉴权、限流、路由
4. 服务层：核心业务逻辑，实验配置管理、分桶计算、事件接收
5. 数据层：MySQL 持久化实验元数据，Redis 缓存配置，Kafka+ClickHouse 处理事件分析
