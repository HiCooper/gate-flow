## 一、平台概述

### 1.1 定位与目标

实验管理平台是AB实验系统的**中枢控制台**，连接分流SDK、数据采集管道、统计分析引擎，为业务方提供从实验创建到决策归档的**一站式实验管理能力**。

**核心目标**:
- **降低实验门槛**: 非技术用户也能自助创建、运行、分析实验
- **保障实验质量**: 内置SRM校验、统计功效计算、护栏监控等质量门禁
- **提升决策效率**: 自动化报告生成、智能决策建议、一键全量/回滚
- **沉淀实验知识**: 实验知识库、经验复用、组织级实验文化积累

### 1.2 平台架构

## 二、实验全生命周期管理

### 2.1 生命周期状态机

```
[草稿 DRAFT] ──▶ [审批 REVIEW] ──▶ [灰度 RAMP 1%-10%] ──▶ [运行中 RUNNING]
                                                                    │
                              ┌─────────────────────────────────────┘
                              │ (自动/手动触发)
                              ▼
                    [分析 ANALYZING] ──▶ [决策 DECISION]
                              │              │
                              │              ├──▶ [全量发布 FULL]
                              │              ├──▶ [迭代优化 ITERATION]
                              │              └──▶ [回滚下线 ROLLBACK]
                              │
                              └─────────────────────────────────────▶ [归档 ARCHIVE]
```

### 2.2 各阶段详细设计

#### 2.2.1 草稿 (DRAFT)

**准入条件**: 无

**核心操作**:
| 功能 | 说明 | 操作人 |
|------|------|--------|
| **实验设计** | 填写实验名称、假设、目标指标 | 产品经理 |
| **流量配置** | 选择域/层、设置流量比例、分组数量 | 产品经理 |
| **指标配置** | 选择主指标、辅助指标、护栏指标 | 数据分析师 |
| **版本配置** | 配置各实验版本的参数差异 | 工程师 |
| **保存草稿** | 随时保存，支持多人协作编辑 | 实验Owner |

**自动校验**:
- 实验名称唯一性检查
- 流量比例合法性（总和=100%，不跨层冲突）
- 指标是否已注册（未注册指标需先走指标申请流程）
- 最小样本量预估（基于历史数据自动计算）

#### 2.2.2 审批 (REVIEW)

**准入条件**: 草稿填写完整，通过自动校验

**审批流程**:
```
实验Owner提交 ──▶ 直属Leader审批 ──▶ 数据科学家审批 ──▶ 平台管理员备案
     │                    │                    │                  │
     │              (业务合理性)        (统计科学性)       (平台合规性)
     │                    │                    │                  │
     │              驳回/通过              驳回/通过            驳回/通过
     │                    │                    │                  │
     └────────────────────┴────────────────────┴──────────────────┘
                                    │
                                    ▼
                              全部通过 → 进入灰度
                              任一驳回 → 退回草稿，附带修改建议
```

**审批检查清单**:
- [ ] **业务合理性**: 实验假设是否清晰？改动是否可解释？
- [ ] **统计功效**: 基于预期效应量和历史方差，计算所需样本量是否可达
- [ ] **流量冲突**: 是否与同层其他实验流量重叠？
- [ ] **护栏指标**: 是否定义了足够的护栏指标保护用户体验？
- [ ] **伦理合规**: 是否涉及敏感用户群体？是否符合数据使用规范？

#### 2.2.3 灰度 (RAMP)

**准入条件**: 审批全部通过

**灰度策略**:
| 阶段 | 流量比例 | 持续时间 | 观察重点 | 自动/手动 |
|------|---------|---------|---------|----------|
| **灰度1** | 1% | 2小时 | SRM检验、崩溃率、核心流程可用性 | 自动 |
| **灰度2** | 5% | 4小时 | 护栏指标趋势、主指标初步方向 | 自动 |
| **灰度3** | 10% | 12小时 | 统计功效达标检查、样本均衡性 | 自动 |
| **全量** | 预设比例 | 实验周期 | 正式实验运行 | 自动/手动 |

**灰度自动门禁**:
```
灰度1通过条件:
├─ SRM检验通过 (p > 0.01)
├─ 崩溃率无显著恶化 (p > 0.05, 单侧)
├─ 核心流程错误率 < 基准值 × 1.5
└─ 无P0级客诉

灰度2通过条件:
├─ 灰度1全部条件持续满足
├─ 主指标方向与预期一致（允许不显著）
└─ 辅助指标无异常波动

灰度3通过条件:
├─ 灰度2全部条件持续满足
├─ 累计样本量达到最小统计功效要求
└─ 自动进入全量运行
```

#### 2.2.4 运行中 (RUNNING)

**核心能力**:
- **实时监控看板**: 主指标、辅助指标、护栏指标的实时趋势图
- **自动告警**: 护栏指标恶化、SRM异常、样本量不足等自动通知
- **中途调整**: 支持流量比例调整（需审批）、实验暂停/恢复
- **样本追踪**: 实时显示各组样本量、预计达到统计功效的时间

**运行期自动操作**:
| 触发条件 | 自动操作 | 通知对象 |
|---------|---------|---------|
| 护栏指标 p < 0.01 (恶化) | 自动暂停实验 | 实验Owner + 值班群 |
| SRM检验 p < 0.01 | 自动暂停 + 标记数据不可信 | 实验Owner + 工程师 |
| 样本量达到预设值 | 推送"可分析"通知 | 实验Owner + 分析师 |
| 实验周期达到上限 | 自动进入分析阶段 | 实验Owner |
| mSPRT早停条件触发 | 推送早停建议 | 实验Owner + 管理层 |

#### 2.2.5 分析 (ANALYZING)

**分析工作流**:
```
实验进入分析 ──▶ 自动报告生成 ──▶ 分析师复核 ──▶ 决策建议生成
                                    │
                                    ├──▶ 统计显著且正向 → 建议全量
                                    ├──▶ 统计显著且负向 → 建议回滚
                                    ├──▶ 不显著但正向 → 建议延长或迭代
                                    └──▶ 不显著且负向 → 建议回滚
```

**自动报告内容**:
| 模块 | 内容 |
|------|------|
| **实验摘要** | 实验名称、周期、样本量、分组比例 |
| **主指标结果** | Lift、CI、p值、统计显著性判断、CUPED修正后结果 |
| **辅助指标结果** | 各辅助指标变化，BH校正后的显著性 |
| **护栏指标结果** | 各护栏指标是否触发恶化告警 |
| **时间趋势** | 按天/小时的主指标变化曲线 |
| **人群拆分** | 按用户属性（新老用户、城市等级、设备类型）的细分分析 |
| **SRM检验** | 分流比例校验结果 |
| **决策建议** | 基于规则的智能建议（全量/迭代/回滚） |

#### 2.2.6 决策 (DECISION)

**决策类型**:
| 决策 | 说明 | 后续操作 | 权限 |
|------|------|---------|------|
| **全量发布 (FULL)** | 实验效果显著正向，推广到全部用户 | 实验配置转为默认配置，进入归档 | 管理层 |
| **迭代优化 (ITERATION)** | 有效果但可优化，基于洞察做新实验 | 生成迭代实验草稿，继承当前配置 | 产品经理 |
| **回滚下线 (ROLLBACK)** | 效果不显著或负向，停止实验 | 全部用户恢复对照组版本，进入归档 | 管理层 |
| **延长实验 (EXTEND)** | 样本量不足，需要继续运行 | 延长实验周期，重新进入运行中 | 数据分析师 |

**决策审批**:
- 全量/回滚：需管理层审批（涉及全站用户影响）
- 迭代：需直属Leader审批
- 延长：需数据科学家确认统计合理性

#### 2.2.7 归档 (ARCHIVE)

**归档内容**:
- 实验完整配置快照
- 原始数据样本（脱敏，保留1年）
- 自动报告PDF
- 决策记录与审批链
- 关联迭代实验链接

**归档后能力**:
- 实验数据只读查询
- 作为后续实验的参照基准
- 知识库自动收录

## 四、权限模型设计

### 4.1 权限设计原则

- **最小权限原则**: 用户仅拥有完成工作所需的最小权限集合
- **职责分离**: 实验创建、审批、分析、决策由不同角色执行
- **可追溯**: 所有权限操作留痕，支持审计
- **动态授权**: 支持临时授权、权限回收、委托代理

### 4.2 RBAC基础角色

| 角色 | 职责 | 典型用户 |
|------|------|---------|
| **超级管理员** | 平台配置、用户管理、全局权限分配 | 平台负责人 |
| **平台管理员** | 指标审核、实验备案、全局监控 | 数据平台团队 |
| **数据科学家** | 实验统计方案审核、复杂分析支持 | 数据科学团队 |
| **部门管理员** | 本部门用户管理、实验审批 | 业务线负责人 |
| **产品经理** | 实验创建、灰度监控、迭代优化 | 产品经理 |
| **数据分析师** | 实验分析、报告生成、洞察挖掘 | 数据分析师 |
| **工程师** | 实验版本开发、SDK集成、技术配置 | 研发工程师 |
| **管理层** | 全量/回滚决策、全局看板查看 | 业务负责人 |
| **访客** | 只读查看实验列表和公开报告 | 新入职/跨部门 |

### 4.4 ABAC动态权限

基于属性的动态访问控制，作为RBAC的补充：

| 属性维度 | 属性示例 | 权限影响 |
|---------|---------|---------|
| **业务线** | 电商/金融/内容 | 只能查看本业务线实验 |
| **数据敏感度** | 公开/内部/机密 | 机密实验仅特定角色可见 |
| **实验状态** | 草稿/运行中/已归档 | 草稿仅参与人可见，归档后公开 |
| **用户职级** | P4/P5/P6/P7+ | 高级别用户自动获得审批权限 |
| **用户组** | 核心项目组成员 | 项目组成员共享实验编辑权限 |

### 4.5 审批流配置

| 实验类型 | 审批链 | 说明 |
|---------|--------|------|
| **常规实验** | 直属Leader → 数据科学家 | 标准流程 |
| **高风险实验** | 直属Leader → 数据科学家 → 部门负责人 | 涉及核心流程或大额流量 |
| **全站实验** | 直属Leader → 数据科学家 → 部门负责人 → 平台管理员 | 影响全站用户体验 |
| **紧急实验** | 事后补审批（24小时内） | 线上故障修复等紧急场景 |

### 4.6 审计与合规

| 审计内容 | 记录粒度 | 保留期限 |
|---------|---------|---------|
| **权限变更** | 谁、何时、变更了什么权限 | 3年 |
| **实验操作** | 谁、何时、对哪个实验做了什么 | 与实验生命周期一致 |
| **数据下载** | 谁、何时、下载了什么数据 | 3年 |
| **审批记录** | 谁、何时、审批结果、意见 | 永久 |
| **登录日志** | 谁、何时、IP、设备 | 1年 |

## 五、界面设计要点

### 5.1 设计原则

- **信息密度**: 实验列表高密度展示，详情页分层展开
- **状态可见**: 实验状态用颜色+图标+文字三重编码
- **操作防错**: 关键操作（全量/回滚）二次确认，不可逆操作明确提示
- **数据可视化**: 指标趋势用折线图，分组对比用柱状图，人群拆分用堆叠图
- **响应速度**: 页面首屏 < 2s，实时数据 < 5s刷新

### 5.2 核心页面

| 页面 | 核心功能 | 设计要点 |
|------|---------|---------|
| **实验列表** | 筛选、排序、批量操作 | 表格+卡片双视图，状态色块 |
| **实验详情** | 配置查看、实时监控、统计报告 | Tab页组织，关键指标置顶 |
| **创建实验** | 分步向导、自动校验、实时预览 | 步骤条+表单校验+流量模拟 |
| **分析看板** | 多维度分析、人群拆分、趋势对比 | 图表联动、下钻分析 |
| **流量地图** | 可视化流量分配、冲突检测 | 树状图+色块比例 |
| **知识库** | 搜索、标签、关联推荐 | 卡片流+搜索高亮 |
