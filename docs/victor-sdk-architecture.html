核心流程
分桶计算流程
1
客户端启动时从服务端拉取实验配置 (REST API)
2
配置存入本地 Caffeine 缓存
3
每秒轮询检查配置版本变化
4
调用 getVariant() 时，本地执行 BucketEngine.computeBucketResult()
5
MurmurHash3(userId#layerId#salt) % 10000 → bucket → 映射变体
设计说明
1. victor-bucketing 模块为零Spring依赖的纯Java模块，可直接嵌入客户端SDK
2. 分桶算法: MurmurHash3 对 userId#layerId#salt 进行哈希，取模10000得到bucket编号 (0-9999)
3. 变体映射: 根据ExperimentSpec中的variantSpecs数组匹配bucket范围 [bucketStart, bucketEnd]
4. 配置更新: 支持增量拉取 (fromVersion参数)，304 Not Modified 时跳过更新
5. 线程安全: 使用ReentrantLock保护配置更新操作
