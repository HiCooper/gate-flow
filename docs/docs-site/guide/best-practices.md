# 最佳实践

本文档总结使用 GateFlow 进行 A/B 测试的最佳实践。

## 实验设计

### 明确实验假设

在开始实验前，明确：
- 为什么要做这个实验？
- 期望的结果是什么？
- 如何衡量成功？

### 选择合适的指标

| 指标类型 | 说明 | 示例 |
|----------|------|------|
| 主指标 | 核心业务指标 | 转化率、GMV |
| 护栏指标 | 防止副作用 | 页面加载时间 |

### 确保足够的样本量

样本量不足会导致：
- 无法检测到真实效果
- 统计结果不可靠

使用[样本量计算器](https://www.statisticalsignificanc calculator.com)预估实验时长。

## 实验执行

### 运行 A/A 测试

在正式实验前，建议：
1. 运行 1-2 天的 A/A 测试
2. 验证分流均匀性
3. 确认指标计算正确

### 监控护栏指标

实验期间持续监控护栏指标，异常时及时停止实验。

### 避免中途修改

- 不要在实验期间修改变体配置
- 不要在实验期间修改流量分配
- 避免多实验交叉影响

## 分析解读

### 统计显著性

- p < 0.05: 结果显著，可以参考
- p >= 0.05: 结果不显著，继续观察或增加样本量

### 效果大小 (Effect Size)

关注实际效果，而不仅是统计显著性：

```
转化率提升 = (treatment - control) / control × 100%
```

### SRM 检查

检查样本分配比例是否与预期一致，检测到 SRM 需排查问题。

## 常见误区

| 误区 | 正确做法 |
|------|----------|
| 只看显著性 | 综合考虑效果大小和业务价值 |
| 实验时间过短 | 运行足够时间，覆盖业务周期 |
| 多重比较 | 使用 BH 校正控制假阳性 |
| peeking | 预设样本量，避免提前停止 |
