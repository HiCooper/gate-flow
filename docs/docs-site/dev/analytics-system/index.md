# 埋点分析系统

GateFlow 埋点分析系统（Tracker）是独立于 AB 实验系统的用户行为数据采集平台，负责采集、存储、分析全站用户行为数据。

## 系统定位

- **独立部署**: 与 AB 实验服务解耦，可独立运维和扩展
- **全面采集**: 覆盖曝光、点击、停留时长、滚动深度、路径等维度
- **高性能**: 支持高并发批量写入，客户端离线队列保障数据不丢失
- **可靠性**: 熔断保护、DLQ 死信队列、Redis 去重

## 与 AB 系统的关系

```
┌──────────────────────────────────────────────┐
│                 客户端 SDK                     │
│  ┌────────────────┐  ┌────────────────────┐  │
│  │  GateFlow SDK  │  │   Tracker SDK      │  │
│  │  实验分流      │  │   行为数据采集      │  │
│  └───────┬────────┘  └─────────┬──────────┘  │
└──────────┼──────────────────────┼─────────────┘
           │                      │
           ▼                      ▼
┌──────────────────┐  ┌────────────────────────┐
│  AB Server       │  │  Tracker Server        │
│  (实验管理/分流)  │  │  (事件接收/存储/聚合)   │
└──────────────────┘  └────────────────────────┘
           │                      │
           └──────────┬───────────┘
                      │  (通过 user_id 关联)
                      ▼
              ┌──────────────┐
              │  ClickHouse  │
              │  (共享存储)   │
              └──────────────┘
```

## 采集维度

| 维度 | 核心指标 | 采集方式 |
|------|---------|---------|
| **页面级** | PV/UV、跳出率、停留时长 | SPA 路由监听、visibilitychange |
| **元素级** | 曝光率、点击率、转化漏斗 | Intersection Observer |
| **路径分析** | 页面序列、关键节点 | 会话级序列聚合 |
| **归因分析** | 渠道贡献、首次/末次触点 | UTM 参数 + 时间窗口 |

## 埋点路径规范

采用四级结构：`a_{应用}.b_{页面}.c_{区块}.d_{点位}`

| 层级 | 前缀 | 示例 |
|------|------|------|
| 应用 | `a_` | `a_app`, `a_admin`, `a_ds` |
| 页面 | `b_` | `b_home`, `b_product`, `b_search` |
| 区块 | `c_` | `c_banner`, `c_recommend`, `c_nav` |
| 点位 | `d_` | `d_btn`, `d_slide_1`, `d_item_3` |

示例：`a_app.b_product.c_action.d_buy_btn` 表示应用→商品详情→操作区→购买按钮。
