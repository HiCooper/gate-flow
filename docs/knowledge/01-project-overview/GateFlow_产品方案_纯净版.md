# GateFlow 产品方案

## 一、产品定位与核心能力

GateFlow 是一款 **AI 驱动的付费墙增长基础设施**，面向移动应用与 Web 产品，提供从付费墙设计、A/B 实验、订阅管理到实时分析的完整变现闭环。通过一次 SDK 集成，团队无需再依赖多平台拼接，即可远程控制付费墙的展示、定价、实验与折扣策略，并实时获取收入数据。

### GateFlow 的核心差异化
1. **全栈覆盖**：移动端（iOS / Android / 跨平台框架）与 Web 端全覆盖，填补 Web SaaS 无成熟付费墙平台的空白。
2. **AI 原生**：内嵌 AI 文案生成、智能实验推荐、多臂老虎机自动优化、动态展示策略，实现从“手动调优”到“AI 自主增长”。
3. **一体化集成**：内置订阅管理与权益引擎，开发者不再需要“付费墙 + 订阅管理”双 SDK，降低系统复杂度。
4. **开发者优先**：提供多环境隔离、本地模拟器、CLI 工具、调试覆盖层与 CI/CD 集成，所有功能对开发者友好、可测试、可复用。
5. **国内可用**：支付宝/微信支付、国内安卓商店适配、中文管理后台、合规数据存储，开箱即用。

## 二、基础能力

GateFlow 的基础能力围绕“编辑 - 实验 - 触发 - 支付 - 分析”完整链路构建，所有功能均面向生产环境打磨，免费层级即可使用全部核心能力：

### 2.1 无代码付费墙编辑器
- **拖拽式可视化构建**，支持实时预览，内置 200 + 付费墙模板，覆盖订阅、一次性购买、试用引导等场景。
- **混合渲染引擎**：可选用原生组件模式获得最佳性能，或采用 Web 技术栈实现更丰富的交互效果；编辑器内所见即所得，无跨平台样式偏差。
- **版本管理**：每次保存自动生成版本快照，支持一键回滚和历史对比。
- **设计系统**：全局控制品牌色、字体、圆角、按钮样式，批量应用到所有付费墙。

### 2.2 高级 A/B 实验平台
- **多变量实验**：支持多变量 A/B/N 测试，可同时测试价格、文案、设计、触发时机等组合。
- **Houldout 对照组**：精确衡量付费墙对整体收入的增量贡献，避免伪增长。
- **深度归因**：实验变体级别的转化、收入、LTV 追踪，数据与实验严格绑定。
- **智能流量分配**：支持固定权重分配和多臂老虎机自动优化，逐步将流量倾向高转化变体。
- **AI 预测与推荐**：基于早期数据预测获胜变体，并自动推荐待测试的变量。

### 2.3 受众定向与动态触发
- **50 + 自动追踪属性**：设备型号、操作系统、网络状态、语言、获客渠道、用户行为频次等。
- **自定义属性**：开发者可上传任意用户画像标签，用于精准定向。
- **行为触发规则**：支持基于用户行为序列的触发条件（如“第 3 次完成核心操作后”）。
- **动态展示策略**：根据用户实时转化概率，自动选择是否展示以及展示哪种付费墙变体。

### 2.4 订阅管理与权益引擎
- **Entitlements 抽象层**：定义功能与权益级别，将其映射到不同应用商店或支付渠道的产品。
- **自动收据验证**：支持 App Store / Google Play / 国内安卓商店的票据即时验证。
- **Web 支付集成**：内置 Stripe、支付宝、微信支付等 Web 支付方式，统一权益管理。
- **退款与欺诈检测**：退款实时回调，可疑交易风控，保障收入真实。
- **跨平台权益同步**：用户在一端购买后，权益自动同步至所有平台，无需额外开发。

### 2.5 实时分析与事件追踪
- **事件驱动架构**：所有付费墙展示、点击、购买等事件近实时采集。
- **秒级更新仪表盘**：转化率、试用开始率、收入等核心指标实时刷新。
- **LTV 预测**：基于早期用户行为数据预测长期价值。
- **用户路径回溯**：可查看个体用户触发付费墙前后行为序列，快速定位问题。
- **行业基准对比**：提供同品类 App 的匿名转化基准作为参考。

### 2.6 跨平台 SDK 与开发者工具
- **全平台 SDK**：iOS（Swift / SwiftUI）、Android（Kotlin / Jetpack Compose）、Web（React / Vue / 原生 JS）、React Native、Flutter。
- **开源与轻量**：核心逻辑统一，平台层面极薄，SDK 体积小，集成仅需几行代码。
- **多环境支持**：Development / Staging / Production 三套环境配置隔离，变更不影响线上。
- **本地模拟器 CLI**：在本地模拟付费墙展示、购买流程、实验分配，缩短开发调试周期。
- **调试覆盖层**：App 内摇一摇即可查看当前付费墙配置、实验分组、缓存状态和网络日志。
- **CI / CD 集成**：提供 GitHub Action 等插件，配置变更可自动预览或校验。

### 2.7 集成与扩展
- **Webhooks**：完整的事件通知系统，可推送到自定义服务或第三方平台。
- **第三方集成**：支持 AppsFlyer、Adjust、Mixpanel、Amplitude、Slack、飞书等。
- **MCP 协议接口**：允许 AI Agent 直接操作付费墙配置和实验管理。
- **Public API & CLI**：供团队构建自定义工具和自动化脚本。

## 三、竞争格局

在付费墙基础设施领域，主要参与者有：

- **Superwall**：聚焦付费墙设计与实验，提供无代码编辑器和深度 A/B 测试，但需要搭配独立订阅管理方案，无 Web 端支持，中国市场适配空缺。
- **RevenueCat**：以订阅逻辑基础设施为主，覆盖收据验证、状态管理等，但付费墙实验能力较弱，且对全量流水收费。
- **Adapty**：合并付费墙与订阅管理，提供较强的分析和模板，但在 Web 端、AI 能力和国内支付适配方面存在短板。

GateFlow 的核心优势在于 **“全端覆盖 + AI 原生 + 一体化整合 + 国内深度适配”**，通过一道 SDK 同时解决付费墙、订阅管理和实验优化问题，显著降低开发团队的碎片化集成成本。

## 四、国内场景专项增强

### 4.1 支付与合规
- **支付渠道**：SDK 内置支付宝、微信支付，支持 App 内购买和 Web 端 H5 / JSAPI。
- **国内应用商店适配**：支持华为、小米、OPPO、vivo 等主流商店的 IAP 票据验证和权益映射。
- **合规助手**：自动生成《用户协议》《隐私政策》模板，满足《个人信息保护法》要求；付费墙内容合规自动审核。

### 4.2 本地化与性能
- **全中文控制台**：编辑器、仪表盘、文档均提供中文，降低团队学习成本。
- **语言与货币**：内置中文模板，支持自动本地化翻译；人民币、港币等多币种定价，实时汇率换算。
- **国内 CDN 与弱网优化**：配置资源通过国内 CDN 分发，弱网下可回退至本地缓存，保证购买流程不中断。
- **数据存储合规**：核心数据存储在符合国内法规的云服务商，满足数据不出境要求。

### 4.3 开发者生态
- **中文社区支持**：提供飞书、微信群即时答疑，中文视频教程和最佳实践案例。
- **国内示例项目**：基于国内主流框架提供完整 Demo，帮助团队快速启动。

## 五、产品架构
```text
┌──────────────────────────────────────┐
│ 应用层 │
│ 编辑器 │ AI实验 │ 受众 │ 分析 │
└──────────────────┬───────────────────┘
┌──────────────────┴───────────────────┐
│ 服务层 │
│ 配置管理 │ 实验分配 │ 权益引擎 │ 事件处理 │
└──────────────────┬───────────────────┘
┌──────────────────┴───────────────────┐
│ 数据层 │
│ 事件流 (Kafka 兼容) │ 分析 (ClickHouse) │ 状态 (PostgreSQL) │
└──────────────────┬───────────────────┘
┌──────────────────┴───────────────────┐
│ SDK 层 │
│ iOS │ Android │ Web │ RN │ Flutter │
└──────────────────────────────────────┘
```

**原则**：SDK 轻量化，逻辑服务端驱动；事件流不丢失可重放；分析实时预聚合；配置秒级下发。

## 六、实施路线图

### Phase 1：MVP（3-4个月）
- iOS、Web SDK，基础配置管理后台，3-5 模板，基础转化仪表盘，权益引擎 v1，本地模拟器 CLI 基础版。
- 支付宝/微信支付（移动端），国内 CDN 部署，中文管理后台 v1。

### Phase 2：实验与增长引擎（4-5个月）
- 可视化编辑器 v1，A/B 测试引擎，Android SDK，50 + 模板，受众定向，Webhooks，多环境隔离，Stripe 及支付宝/微信 Web 支付。

### Phase 3：AI 智能化（4-6个月）
- 中文 AI 文案生成，智能实验推荐，多臂老虎机自动优化，AI 动态展示，行业基准对比，React Native / Flutter SDK。

### Phase 4：生态与平台化（持续）
- 模板市场（支持人民币交易），MCP 协议接口，企业私有化部署，国内全方位社区运营。
